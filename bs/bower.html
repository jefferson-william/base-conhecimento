Comments are not allowed in the bower.json file (See http://blog.getify.com/json-comments/ for why this is silly), so they are added here. The bower.json file is responsible for the installing the styles (CSS, LESS, etc.) and scripts (JavaScript, TypeScript etc.) using Bower. See the official bower.json specification for more information.
  • name - This is used if you want to create your own Bower package. Unfortunately, this is required by Bower. It does not matter what goes here too much.
  • private - This is not a public bower repository This is a way to prevent accidental publication of private repositories.
  • dependencies - The client side styles and scripts your application depends on. These are downloaded to a bower_components folder at the root of the project.
